Module 8: Game Theory

Core Definitions

  • Normal (Strategic) Form: Simultaneous games represented as a payoff matrix
  • Extensive Form: Sequential games represented as a game tree with decision nodes
  • Dominant Strategy: Yields the highest payoff for a player regardless of what the opponent does. If it exists, always play it.
  • Dominated Strategy: Always yields a lower payoff than another strategy for every opponent action. Never play it.
  • Best Response: The strategy that maximizes a player's payoff given the opponent's strategy. BRi(s-i) = argmax πi(si, s-i)
  • Nash Equilibrium (NE): A pair of strategies where BOTH players are playing their best responses simultaneously. No player wants to unilaterally deviate. A game can have 0, 1, or multiple NEs.

How to Solve Normal Form Games

  1. Step 1: Check for dominant strategies (best regardless of opponent)
  2. Step 2: Eliminate dominated strategies (iterated elimination)
  3. Step 3: Find best responses: for each column, find Row player's max payoff (underline). For each row, find Column player's max payoff (underline).
  4. Step 4: NE = any cell where BOTH payoffs are underlined (both playing best response)

Classic Game Types

Prisoners' Dilemma

CooperateDefect
Cooperate(-1, -1)(-9, 0)
Defect(0, -9)(-6, -6) NE

Both defect even though mutual cooperation is better. Defect is dominant for both.

Battle of the Sexes (Coordination)

Two NEs exist, players prefer different ones. Coordination is the challenge.

Matching Pennies

No pure strategy NE exists. Only mixed strategy equilibrium.

Sequential Games

  • Solve by backward induction: start at terminal nodes, work backward
  • At each node, the player picks the action giving them the highest payoff
  • Value of commitment = Payoff in sequential game - Payoff in simultaneous game
  • First mover can manipulate the follower's incentives

Key Formulas

Nash Equilibrium condition: s₁* = BR₁(s₂*) AND s₂* = BR₂(s₁*) Best Response: BR_i(s_{-i}) = argmax π_i(s_i, s_{-i}) Value of Commitment: V = π_sequential - π_simultaneous

Problem 5HARD

Design Your Own Payoff Matrix

Two firms choose High or Low price. Fill in payoff matrix values such that:

  • The game is a Prisoners' Dilemma (both prefer mutual High, but Low is dominant for both)
  • Monopoly profit (both High) = $100 each
  • If one undercuts while other stays High, undercutter gets $150

(a) What range of values for the (Low, Low) payoff makes this a valid Prisoners' Dilemma?

(b) What is the NE?

Solution

HighLow
High(100, 100)(X, 150)
Low(150, X)(Y, Y)

(a) For Low to be dominant: Y > X (Low beats High when opponent plays Low) and 150 > 100 (already satisfied). For PD: need (100, 100) Pareto-dominates (Y, Y), so Y < 100. Also need Y > X. Also for Low to dominate High when opponent plays Low: Y > X. The "sucker" payoff X must be < Y. So we need: X < Y < 100. Example: X = 20, Y = 60.

(b) NE: (Low, Low) with payoffs (Y, Y) — both playing dominant strategy despite mutual High being better.

Module 9: Price Competition (Bertrand)

The Bertrand Model

  • Firms compete by setting prices (not quantities)
  • With identical goods and identical MC: any price above MC gets undercut
  • Bertrand Trap: NE is P = MC, profit = 0 — even with just 2 firms!
  • This is called the "Bertrand Paradox" — 2 firms produce competitive outcome

Five Ways to Escape the Bertrand Trap

  1. Cost leadership: Lower MC → price just below rival's MC, capture whole market
  2. Product differentiation: Make goods non-identical in variety or quality. With differentiation, firms have some pricing power (P > MC)
  3. Limit production capacity: If you can't serve the whole market, rival has no reason to undercut to zero
  4. Price matching: "We'll match any price" removes incentive to undercut
  5. Long-term interaction: Repeated game enables tacit cooperation (see Module 10)

Differentiated Bertrand: Solving Best Response

Given demands: q₁ = a - p₁ + b·p₂ and q₂ = a - p₂ + b·p₁ Marginal cost = c for both firms Firm 1's profit: π₁ = (p₁ - c)(a - p₁ + b·p₂) FOC: dπ₁/dp₁ = (a - p₁ + b·p₂) + (p₁ - c)(-1) = 0 a - 2p₁ + b·p₂ + c = 0 p₁* = (a + c + b·p₂) / 2 ← Best Response for Firm 1 By symmetry: p₂* = (a + c + b·p₁) / 2 Solving simultaneously: p₁* = p₂* = (a + c) / (2 - b)

Time vs Newsweek Example

qT = 5 - pT + 0.5pN, MC = $1

BR_T: p_T* = 3 + 0.25·p_N BR_N: p_N* = 3 + 0.25·p_T Solving: p_T* = p_N* = $4 NE prices = $4 each Compare: Monopolist owning both → p = $5.50 each (higher)

Cost Leadership Example: Senstrum

Your MC = $2/dose. Foreign rival MC = $3.02/dose. Demand: Q = 400 - 50P

Strategy: Price just below rival's MC → P = $3.02 Q = 400 - 50(3.02) = 248.95 million doses Profit = (3.02 - 2) × 248.95 = $254 million

Differentiated Duopoly

Two coffee chains. Demands: q₁ = 100 - 2p₁ + p₂ and q₂ = 100 - 2p₂ + p₁. Both have MC = $10.

(a) Derive each firm's best response function.

(b) Find the Nash Equilibrium prices and quantities.

(c) Calculate each firm's profit at NE. Compare to the monopoly outcome if one firm owned both.

(a) π₁ = (p₁ - 10)(100 - 2p₁ + p₂)

dπ₁/dp₁ = (100 - 2p₁ + p₂) + (p₁ - 10)(-2) = 0

100 - 2p₁ + p₂ - 2p₁ + 20 = 0 → 120 + p₂ = 4p₁

BR₁: p₁ = (120 + p₂)/4 = 30 + p₂/4

By symmetry: BR₂: p₂ = 30 + p₁/4

(b) p₁ = 30 + (30 + p₁/4)/4 = 30 + 7.5 + p₁/16

(15/16)p₁ = 37.5 → p₁* = p₂* = $40

q₁ = 100 - 2(40) + 40 = 60

(c) π₁ = (40-10)(60) = $1,800 each

FOC: 2(100-p) + 2(p-10)(-1) = 0 → 100-p-p+10 = 0 → p = $55

Problem 3HARD

Cost Leadership with Exchange Rate Risk

You sell a drug with MC = $2/dose. Demand: Q = 400 - 50P (millions). A foreign rival has MC = INR 145 with exchange rate ε (INR per USD).

(a) If ε = 48 (INR/USD), what is your Bertrand equilibrium profit?

(b) An $80M ad campaign would increase demand by 40%. Should you do it?

(c) If the rupee appreciates to ε = 40, how does this affect your profit (without the campaign)?

Solution

(a) Rival's MC in USD = 145/48 = $3.021. You price just below $3.021.

Q = 400 - 50(3.021) = 248.95M. π = (3.021 - 2)(248.95) = $254M

(b) New demand: Q = 1.4(400 - 50P) = 560 - 70P

Q = 560 - 70(3.021) = 348.53M. π = (1.021)(348.53) = $355.8M

Increase = 355.8 - 254 = $101.8M > $80M cost. Yes, do the campaign.

(c) Rival's MC = 145/40 = $3.625. Higher rival cost → you can charge more.

Q = 400 - 50(3.625) = 218.75M. π = (3.625 - 2)(218.75) = $355.5M (higher!)

Module 10: Cooperation & Collusion

Explicit Collusion (Cartels)

  • Firms agree to restrict output to "quotas" → price rises → higher joint profits
  • Result: maximize total profits, like a shared monopoly
  • Cartels are strictly illegal — felonies in US, EU, Japan, Korea, Brazil
  • Each member has incentive to cheat (produce more at the high price)
  • Enforcement: if cheater detected, cartel punishes (price war)

Repeated Games and Trigger Strategies

When the same game is played infinitely, cooperation can be sustained as a Nash Equilibrium:

TRIGGER STRATEGY: "Play high price as long as all others played high price in all prior periods. Play P = MC forever if anyone ever deviated." COOPERATION CONDITION (n firms, interest rate r): NPV(cooperate) = π_mono/n + (π_mono/n)/r NPV(deviate) = π_mono + 0 (punished forever after) Cooperate when: π_mono/n + (π_mono/n)/r > π_mono Simplifies to: r < 1/(n-1) EXAMPLES: 2 firms: r < 1 (almost always cooperate) 3 firms: r < 1/2 (cooperate if r < 50%) 4 firms: r < 1/3 (cooperate if r < 33%)

With Antitrust Enforcement

Detection probability X after period 1, penalty = k × profits so far E[cartel] = (1-X) × [π_coop + π_coop/r] + X × [π_coop - k×π_coop] = (1-X) × [π_coop(1 + 1/r)] + X × [π_coop(1 - k)] Compare to: NPV(deviate) = π_mono Solve for maximum X at which cartel is preferred.

Finite Repeated Games

  • Cooperation UNRAVELS via backward induction
  • Last period: no future punishment → everyone deviates
  • Second-to-last: knowing last period is deviate → deviate now too
  • Result: NE (P = MC) in EVERY period

Tacit Collusion

  • No explicit agreement → NOT illegal
  • Firms independently choose trigger strategies → cooperation is NE
  • Easier with: fewer firms, lower interest rates, transparent pricing, repeated interaction
Problem 1HARD

Three-Firm Repeated Bertrand (from Spring 2025 Final)

3 firms, MC = 100, demand Q = 1000 - P, interest rate r = 5%.

(a) What is the one-period NE?

(b) Can cooperation be sustained with trigger strategies in the infinite game? Show NPV calculations.

Solution

(a) Bertrand with identical goods → NE: P = MC = 100, all profits = 0

(b) Monopoly: MR = 1000 - 2Q = MC = 100 → Q = 450, P = 550

πmono = (550-100)(450) = $202,500

Each firm's share: 202,500/3 = $67,500

NPV(cooperate) = 67,500 + 67,500/0.05 = 67,500 + 1,350,000 = $1,417,500

NPV(deviate) = 202,500 (undercut, grab whole market for 1 period, then 0 forever)

1,417,500 >> 202,500. Yes, cooperation is sustainable.

Check: r = 5% < 1/(3-1) = 50%. Condition satisfied.

Problem 3HARD

Finite Game Unraveling

Same 4 firms (P = 2000 - Q, MC = 0) play the pricing game for exactly 3 periods. Each firm uses the trigger strategy.

(a) What price is charged in period 3?

(b) What price is charged in period 2?

(c) What price is charged in period 1?

(d) What is each firm's total profit across all 3 periods?

Solution

(a) Period 3: Last period → no future punishment. All firms undercut. P₃ = MC = 0

(b) Period 2: Period 3 is P=0 regardless, so no future cooperation to protect. P₂ = MC = 0

(c) Period 1: Same logic — periods 2 and 3 are already P=0. P₁ = MC = 0

(d) Total profit = $0 for every firm in every period.

Key insight: With finite horizon, trigger strategies collapse via backward induction. The punishment mechanism requires infinite future to work.

Module 11: Commitment & Entry Deterrence

Core Concepts

  • Commitment: An irreversible action that constrains your future choices to manipulate rival's behavior
  • Must be visible (rival sees it), credible (in your interest to follow through), and irreversible
  • Cortés burned his ships → committed to fight → natives surrendered
  • Building excess capacity → signals you'll flood market if rival enters

Entry Deterrence

  • Incumbent can invest to make entry unprofitable for rival
  • Key insight: Investment may lower your payoffs in ALL scenarios but changes rival's decision
  • Compare: π(invest, rival stays out) vs π(no invest, rival enters)

Solving Sequential Games

BACKWARD INDUCTION: 1. Start at terminal decision nodes 2. At each node, player picks action with highest payoff 3. Replace node with that payoff 4. Work backward to root FIRST-MOVER ADVANTAGE: Exists when sequential payoff > simultaneous payoff Value = π_sequential - π_simultaneous

Preemption Strategies

  • Predatory pricing: Price below cost to drive rival out (legally risky)
  • Product proliferation: Fill product space so rival can't find a niche
  • Excessive capacity: Signal willingness to flood market
  • Learning curve: Early production lowers future costs
Problem 1HARD

Computer Chip Capacity Investment (from Practice Problems)

You're the only producer of a specialized chip. A consultant says "don't invest" because payoffs are lower with investment in all scenarios:

Rivals EnterRivals Don't Enter
No Investment($20M, $10M)($50M, $0)
Investment($10M, -$2M)($40M, $0)

(a) Draw the game tree (you move first, rivals observe and decide).

(b) Is the consultant right? Use backward induction.

Solution

Backward induction:

If you invest: Rivals compare Enter (-$2M) vs Don't Enter ($0) → Don't Enter

If you don't invest: Rivals compare Enter ($10M) vs Don't Enter ($0) → Enter

You compare: Invest + No Entry ($40M) vs No Invest + Entry ($20M)

Invest! You get $40M vs $20M. The consultant is WRONG.

The investment's value isn't in the direct payoff comparison — it's in deterring entry. Even though payoffs are lower in every scenario, the investment changes which scenario occurs.

Module 12: Asymmetric Information

Three Types of Information Problems

1. Adverse Selection (Hidden Type) — BEFORE the transaction

  • One party knows their type/quality; the other doesn't
  • Market for Lemons (Akerlof):
  • Buyer offers price of average quality → high-quality sellers exit → average falls → more exit → market unravels to only lowest quality
  • This is a market failure — efficient trades don't happen
LEMONS SPIRAL: 1. Qualities range from q=0 (bad) to q=1 (good) 2. Buyer offers price for average quality: p(q̄₁) where q̄₁ = 0.5 3. Sellers with q > q̄₁ leave (their car is worth more) 4. Remaining average drops to q̄₂ < q̄₁ 5. More sellers leave... 6. Market unravels → only q=0 cars remain

2. Moral Hazard (Hidden Action) — AFTER the transaction

  • One party takes unobservable actions that affect the other
  • Examples: reckless driving after buying insurance, employee shirking, "too big to fail" banks taking excessive risk
  • Remedies: monitoring, incentive pay (stock options, bonuses), deductibles, no-claim discounts, reputation

3. Signaling vs Screening

SignalingScreening
Who acts?Informed partyUninformed party
GoalReveal own typeSort other's types
ExamplesWarranties, education, certificationsInsurance menus, job tests
Key conditionSignal must be costlier for low typeContracts must be incentive-compatible

Health Insurance & ACA

  • Without mandate: healthy people opt out → pool gets sicker → premiums rise → more opt out → spiral
  • ACA individual mandate forces healthy people into pool, preventing adverse selection spiral
Problem 1HARD

Used Car Market (from Spring 2025 Final)

Explain how adverse selection leads to market failure in the used car market. Include: (a) the mechanism of market unraveling, (b) what the seller can do to fix it.

Solution

(a) Market unraveling mechanism:

  1. Sellers know their car's quality; buyers don't
  2. If quality ranges from q=0 to q=1, buyers offer price for average quality p(0.5)
  3. Sellers with q > 0.5 withdraw (their cars are worth more than offered)
  4. Remaining cars have average quality < 0.5, say q̄ = 0.25
  5. Buyers adjust offer down to p(0.25)
  6. More sellers withdraw...
  7. Only lowest quality cars (q = 0) remain — market failure

(b) Seller solutions:

  • Warranties/guarantees: Signal quality (costly for low-quality to mimic)
  • Third-party certifications/inspections
  • Reputation building
  • Money-back guarantees
  • Disclosure of maintenance records
Problem 2HARD

Suits R Us — Litigation Finance

SRU (an investment firm) finances lawsuits for a percentage of proceeds. SRU has better information about case quality than the plaintiff.

(a) What type of asymmetric information problem is this? Who is informed?

(b) What should the plaintiff worry about?

Problem 3HARD

Business Sale with Asymmetric Information

A business seller values the company uniformly between $100M and $190M. A buyer would gain $10M in synergies (so the business is worth seller's value + $10M to the buyer). The buyer must make a take-it-or-leave-it offer.

If buyer offers $X, the seller accepts if their value ≤ $X.

(a) If buyer offers $130M, what is the probability of acceptance?

(b) Given acceptance, what is the expected value of the business to the seller (and thus to the buyer with synergies)?

(c) What is the buyer's expected profit from offering $130M?

(d) What is the optimal offer?

Solution

(a) Seller values uniformly $100M-$190M. Offer $130M accepted when seller value ≤ $130M.

P(accept) = (130-100)/(190-100) = 30/90 = 1/3 ≈ 33%

(b) Given acceptance, seller value is uniform [$100M, $130M]. Expected seller value = (100+130)/2 = $115M.

Business worth to buyer = $115M + $10M = $125M

(c) E[profit] = P(accept) × (value to buyer - offer) = (1/3)(125 - 130) = (1/3)(-5) = -$1.67M (negative!)

Due to adverse selection: when seller accepts a high offer, it's because the business is worth less.

(d) Trying lower offers: At offer = $110M, P(accept)=10/90=1/9, E[seller value|accept]=$105M, value to buyer=$115M, E[profit]=(1/9)(115-110)=$0.56M. At $100M, P(accept)→0. Optimal offer is around $100-110M — adverse selection forces very low offers.

Problem 4HARD

Moral Hazard Identification

For each scenario, identify whether it's adverse selection, moral hazard, or both. Explain who is informed, what's hidden, and suggest a remedy:

  1. A bank lends to a startup that then gambles with the funds
  2. A health insurer can't tell if applicants are healthy or sick
  3. A manager gets paid a salary regardless of company performance
  4. A "too big to fail" bank takes on excessive derivative positions

Solution

  1. Moral hazard. Bank can't observe startup's actions after lending. Hidden action: risky gambling. Remedy: milestone-based disbursement, monitoring, covenants.
  2. Adverse selection. Insurer (uninformed) can't distinguish types. Sick people (informed) more likely to buy. Remedy: screening (deductible menus), mandatory coverage (ACA mandate).
  3. Moral hazard. Shareholders can't observe manager's effort. Hidden action: shirking. Remedy: stock options, performance bonuses, board oversight.
  4. Moral hazard. Government guarantee removes downside risk. Hidden action: excessive risk-taking. Remedy: regulation, capital requirements, stress tests, limiting bank size.

Module 13: Platforms & Network Effects

Two Types of Network Effects

  • Direct (Same-Side): More users of the SAME type increase value for each user. Examples: telephone, Facebook, fax machine
  • Indirect (Cross-Side): More users on one SIDE increase value for the OTHER side. Examples: Uber (riders ↔ drivers), Xbox (gamers ↔ developers), AmEx (cardholders ↔ merchants)

Key Formulas

Network connections with n users: n(n-1)/2 2 phones: 1 connection 5 phones: 10 connections 12 phones: 66 connections Platform pricing (AmEx example): Merchant fee: +3% per transaction Cardholder reward: -1% (cashback) Platform net revenue: 2% per transaction $100 purchase: Merchant pays: $103 Cardholder pays: $99 AmEx keeps: $2

Winner-Take-Most

  • Strong network effects + incompatibility → market tips to one platform
  • Why "most" not "all"? Multi-homing possible, niche segments, regulation
  • Critical mass: Minimum users needed for platform to be self-sustaining
  • Example: VHS beat Betamax despite arguably lower quality — network effects dominated

Platform Pricing Strategy

  • Subsidize the more elastic/harder-to-get side, charge the less elastic side
  • Total surplus maximizing price can be below MC when network effects exist (subsidy is efficient)
  • Platforms achieve highest market capitalizations because network effects create strong moats

Interoperability

  • With interoperability: Multiple platforms coexist (email: Gmail ↔ Outlook ↔ Yahoo)
  • Without interoperability: Tipping toward monopoly (Xbox games ≠ PlayStation games)
  • Incumbent typically opposes interoperability; entrant favors it

Privacy Issues

  • Platforms collect vast user data → privacy concerns
  • Data is the "price" users pay for free services
  • Network effects lock users in, reducing bargaining power on privacy
Problem 1HARD

Network Effects Analysis (from Spring 2025 Final)

(a) Define the two kinds of network effects. Give an example of each.

(b) A new social media platform has 10,000 users. A competing platform has 1,000,000 users. Both are incompatible. Explain using network effects theory why the smaller platform is likely to fail.

(c) Would mandating interoperability help or hurt the smaller platform? Explain.

Solution

(a)

  • Direct (same-side): User values more users of the same type on the platform. Example: Facebook — you want your friends to be on it too.
  • Indirect (cross-side): User on one side values more users on the other side. Example: Uber — riders want more drivers (shorter wait); drivers want more riders (more business).

(b) With incompatibility, network effects create a winner-take-most dynamic. The larger platform (1M users) has vastly more value per user: connections = n(n-1)/2. For 1M users: ~500 billion connections vs 10K users: ~50 million connections. Users will migrate to the larger network. Advertisers prefer the larger audience. Content creators go where the audience is. The smaller platform fails to reach critical mass and collapses.

(c) Interoperability would HELP the smaller platform. If users on the small platform can connect with users on the large platform, the small platform's users get the benefit of the large network. This eliminates the winner-take-most dynamic and allows both to coexist (like email). The small platform can compete on features/privacy rather than network size.

Problem 2HARD

Platform Pricing Strategy

A new ride-sharing platform has these groups: 1 million potential riders (willing to pay $5/ride) and 100,000 potential drivers (need at least $3/ride to participate). Each side's participation depends on the other's.

(a) Which side should the platform subsidize? Why?

(b) Why might the total surplus maximizing price be below marginal cost?

(c) Explain why this platform might adopt a "winner-take-most" strategy despite short-term losses.

Solution

(a) Subsidize drivers (the scarcer/harder-to-get side). Drivers are the bottleneck — there are 10x more potential riders than drivers. More drivers → shorter wait times → more riders join → more business for drivers (cross-side network effects). Charge riders (the less elastic/more abundant side).

(b) With network effects, each additional user creates value for ALL other users (positive externality). The social benefit of adding a user exceeds their private willingness to pay. To maximize total surplus, you subsidize participation below MC because the network externality makes each user more valuable than their individual payment suggests.

(c) Due to network effects and incompatibility, the market tends toward one dominant platform. The platform that reaches critical mass first triggers a positive feedback loop: more riders → more drivers → even more riders → market tips in their favor. Short-term losses are an investment in reaching critical mass before competitors. Once dominant, the platform can raise prices (switching costs + network lock-in protect the position).

(b) Monopoly: MR = 500 - 2Q = 100 → Q = 200, P = 300

Comprehensive 2HARD

Multi-Topic: Platform with Adverse Selection

An online marketplace connects buyers and sellers of antiques. Sellers know quality (High or Low); buyers don't.

  • 50% of antiques are High quality (worth $1000 to buyer, $800 to seller)
  • 50% are Low quality (worth $200 to buyer, $100 to seller)

(a) If buyers can't distinguish quality, what price do they offer? Which sellers accept? Does the market unravel?

(b) The platform introduces a certification service (costs $50, only profitable for high-quality sellers to use). How does this solve the adverse selection problem?

(c) The platform has direct network effects (buyers value more buyers for price discovery) and indirect network effects (buyers ↔ sellers). Which type of network effect is more important for this marketplace? Why?

Solution

(a) Buyers offer E[value] = 0.5(1000) + 0.5(200) = $600.

High-quality sellers accept ($600 > $800)? No! $600 < $800. They withdraw.

Only low-quality sellers remain. Buyers know this → offer $200. Only low quality trades. Market unravels — high-quality antiques never sell.

(b) This is a signaling/screening mechanism:

  • High-quality sellers: pay $50 for certification. Buyers trust certified items → pay $1000. Seller profit: $1000 - $800 - $50 = $150 > 0. Worth it!
  • Low-quality sellers: if they certify, buyers pay $1000 but item is worth $200 → caught, reputation destroyed. Or: certification detects quality. Either way, not worth faking.

Result: Separating equilibrium. Certified = high quality ($1000), uncertified = low quality ($200). Both types trade!

(c) Indirect (cross-side) network effects are more important. Buyers want more sellers (more selection), sellers want more buyers (more demand/better prices). This is the core value proposition. Direct effects (buyer-buyer) are secondary — buyers benefit from other buyers mainly through better price discovery, but the primary draw is having many sellers to choose from.

QUESTION 1 (20 points)

Q1(a)5 PTS

Platform Pricing

What is a "platform"? Explain the difference between a platform's pricing strategy and a traditional firm's pricing strategy. Why might a platform rationally charge one side of the market below marginal cost?

Solution

A platform is a business that creates value by facilitating interactions between two or more distinct groups of users (e.g., buyers and sellers, riders and drivers).

Traditional firm: Sets P ≥ MC for its product. Platform: Sets different prices for each side. The platform subsidizes the more elastic/harder-to-get side and charges the less elastic side.

A platform may rationally charge below MC because of network externalities: each additional user on the subsidized side creates value for users on the other side (indirect/cross-side network effects). The social benefit of adding that user exceeds their private willingness to pay. The subsidy is recouped from the other side, which values the larger network.

Example: AmEx charges merchants +3% but gives cardholders -1% cashback. The net 2% is AmEx's revenue. Subsidizing cardholders attracts more cardholders, which makes the platform more valuable to merchants.

For (25%, 25%) to be a NE, both firms must be playing best responses.

For 25% to be GrubRun's best response to DashDoor's 25%, we need 6 ≥ 7... wait: GrubRun compares its payoffs in the 25% row for DashDoor: at GR:15% gets 5, at GR:25% gets 6, at GR:35% gets 3. DashDoor at 25%: DashDoor gets W, compared to 8 (at DD:15%) and 9 (at DD:35%) — these are the payoffs in the GrubRun:25% column. For DashDoor's 25% to be a best response, we need W ≥ 8 and W ≥ 9.

So we need W ≥ 9. But the question asks for the highest value of W for which this is an NE. Any W ≥ 9 works. There's no upper bound from the NE conditions.

However — if W is too large, (25%, 25%) is still an NE, but we need to check whether other NEs emerge. The question asks only whether (25%, 25%) is a NE. If the question is asking "highest W such that (25%, 25%) is the unique NE," the answer would require further analysis. As stated: There is no highest value — any W ≥ 9 works. Note: This is a trick — Economides likes to see if you'll recognize when a question's framing doesn't quite work. The key insight is showing the NE conditions: W ≥ 9.

QUESTION 3 (40 points)

Suppose an industry has demand P = 1500 - Q, and all firms have a constant marginal cost of $300 and zero fixed costs.

Q3(a)5 PTS

Monopoly

Find the monopoly price, quantity, and profits.

Solution

P = 1500 - Q → MR = 1500 - 2Q MR = MC: 1500 - 2Q = 300 → Q* = 600 P* = 1500 - 600 = $900 π = (900 - 300)(600) = $360,000
Q3(b)5 PTS

Three-Firm Bertrand

Suppose three firms compete by setting prices in a simultaneous game for this homogeneous good. What prices do the firms set in the Nash equilibrium and why?

Solution

With identical homogeneous goods and identical MC = $300, Bertrand competition drives prices down to marginal cost.

Any price P > 300: a rival can undercut by $0.01 and capture the entire market. This undercutting continues until no firm can profitably undercut further.

NE: P₁ = P₂ = P₃ = $300, profit = $0 for all firms.

This is the Bertrand Paradox — just 3 firms produce the perfectly competitive outcome.

Q3(c)5 PTS

Cost Leadership Escape

One firm develops a proprietary process that reduces its marginal cost to $200 while the other two firms remain at MC = $300. In the new Bertrand equilibrium, what price does the cost leader charge, what quantity does it sell, and what is its profit?

Solution

The cost leader prices just below the rivals' MC to capture the entire market:

P = $300 (just below, effectively $299.99 ≈ $300) Q = 1500 - 300 = 1,200 π = (300 - 200)(1200) = $120,000

The rivals cannot profitably match this price (it's at or below their MC). The cost leader captures the entire market and earns $120,000.

Note: The cost leader could earn even more as a monopolist (set MR = MC = 200 → P = $850, Q = 650, π = $422,500), but can't do that because the rivals would undercut at any price above $300.

Q3(d)5 PTS

Cartel — Interest Rate Condition

Suppose instead all three firms have MC = $300 again, and they form a cartel, splitting monopoly profits equally. There is no antitrust law. The interest rate is R. For what values of R will each firm prefer to stay in the cartel rather than undercut?

Solution

Monopoly profit = $360,000 (from part a) Each firm's cartel share = 360,000/3 = $120,000 per period NPV(cooperate) = 120,000 + 120,000/R NPV(deviate) = 360,000 + 0 (grab whole market, then punished forever) Cooperate when: 120,000 + 120,000/R > 360,000 120,000/R > 240,000 R < 120,000/240,000 R < 1/2 = 50%

Check with formula: R < 1/(n-1) = 1/(3-1) = 1/2. ✓

Firms prefer the cartel when R < 50%.

Q3(e)5 PTS

Antitrust Enforcement

Now suppose there IS antitrust enforcement. The probability of detection after the first period is X. If detected, cartel members pay a penalty equal to twice the monopoly profits they have realized so far, and are forced to the one-period NE forever after. The interest rate is 10%. For what values of X will firms still participate in the cartel?

Solution

Each firm's cartel share per period = $120,000. R = 10%. If NOT detected (prob 1-X): Get 120,000 + 120,000/0.10 = 120,000 + 1,200,000 = $1,320,000 If detected (prob X): Get 120,000 - 2×120,000 = 120,000 - 240,000 = -$120,000 (plus $0 forever after) E[cartel] = (1-X)(1,320,000) + X(-120,000) = 1,320,000 - 1,320,000X - 120,000X = 1,320,000 - 1,440,000X NPV(deviate) = 360,000 Cartel when: 1,320,000 - 1,440,000X > 360,000 960,000 > 1,440,000X X < 960/1440 = 2/3 ≈ 66.7%

Note: With only a 2× penalty (vs 3× in the 2025 final), the cartel is more resilient to detection. This shows how penalty severity matters for deterrence.

Q3(f)5 PTS

Finite Game Unraveling

Suppose instead the game is played for exactly four periods (finite). What is the optimal pricing strategy in period 4? Period 3? Period 1? What are total profits across all four periods?

Solution

Period 4 (last): No future punishment possible → everyone undercuts → P = MC = $300, profit = $0

Period 3: Everyone knows period 4 is P = $300 regardless. No future cooperation to protect → P = $300, profit = $0

Period 2: Same backward induction. Periods 3 and 4 are $300 regardless → P = $300, profit = $0

Period 1: Same logic → P = $300, profit = $0

Total profit = $0 for every firm across all four periods.

Key insight: With a finite horizon, cooperation unravels completely via backward induction. The trigger strategy requires an infinite future to work — the threat of "punish forever" is meaningless when "forever" has an end date.

  • If Don't Invest: Entrant compares $15K vs $0 → Enters. You get $120K.

You compare: $130K (invest) vs $120K (don't invest).

Invest! You get $130K vs $120K. The consultant is WRONG.

The consultant is correct that investment lowers payoffs in every scenario ($130K < $180K, $70K < $120K). But the investment changes which scenario occurs. Without investment, you're stuck in the bad scenario (entry). With investment, you move to the good scenario (no entry). The value of commitment = $130K - $120K = $10K.

Q3(h)5 PTS

Adverse Selection in Hiring

The cost leader also faces an adverse selection problem when hiring engineers: highly skilled engineers (worth $200K/year) and low-skilled ones (worth $80K/year) both apply. The firm can't tell them apart. If the firm offers a salary equal to the expected value (assuming 50/50 mix), explain the market failure that results. Name two mechanisms the firm could use to solve this problem, and identify whether each is signaling or screening.

Solution

Market failure (lemons spiral):

  1. Firm offers average salary: (200K + 80K)/2 = $140K
  2. Highly skilled engineers (worth $200K) won't accept $140K — they can get better offers elsewhere. They withdraw.
  3. Only low-skilled engineers (worth $80K) remain in the pool.
  4. Firm realizes this and adjusts offer down to $80K.
  5. Result: Only low-skilled engineers get hired — market failure. The firm misses out on high-skilled talent, and high-skilled engineers miss out on the job. Efficient trades don't happen.

Solutions:

  • Require a certification or degree (e.g., advanced CS degree, professional credential). This is signaling — the informed party (engineer) takes a costly action to reveal their type. Works because the credential is harder/costlier for low-skilled engineers to obtain.
  • Offer a menu of contracts: (a) lower base salary + high performance bonus, (b) higher base salary + no bonus. High-skilled engineers self-select into (a) because they're confident in performance. This is screening — the uninformed party (firm) designs options that sort types through self-selection. Incentive compatibility ensures each type picks the "right" contract.
